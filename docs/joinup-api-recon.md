# Разведка источника: Join Up Baltic (joinup.lv)

> Дата разведки: 2026-07-06. Метод: статический анализ JS-бандлов Nuxt-приложения + живые пробы через curl.
> **Вывод: за сайтом есть чистый JSON API. Скрапинг HTML не нужен. Сложность v0 — низкая.**

## Общее

- Рабочий домен: `https://joinup.lv` (редирект на `/lv`). `joinupbaltic.eu` — мёртв (522).
- Фронтенд: Nuxt 3 SPA. Все данные ходят через JSON API.
- База API: `https://joinup.lv/api/main/<endpoint>` (прокси на их бэкенд).
- CMS (новости, баннеры): `https://joinup.lv/api/cms` (Strapi) — нам не нужна.
- Картинки отелей: `https://media.joinup.travel/storage/hotel/...`.
- Аутентификация для чтения **не нужна**: обычные GET, без CSRF, без cookies. Обычный `User-Agent` достаточен.
- API дружелюбно валидирует: при неверных параметрах возвращает JSON с `errors[].field` — сам подсказывает имена полей.
- Общие query-параметры везде: `lang=lv` (или `en`, `ru`), `currency=EUR`.

## Эндпоинты (проверены живьём)

### `GET search/origins`
Города вылета. **Рига = `3164`**, Таллин = `2552`, Вильнюс = `2151`.

### `GET tour/destinations?origins=3164`
Направления из Риги + минимальная цена. На дату разведки — 8 стран:

| id | Страна | price_from (EUR) |
|---|---|---|
| c_8 | Турция | 377.50 |
| c_4 | Греция | 424.00 |
| c_28 | Черногория | 438.00 |
| c_3 | Болгария | 439.00 |
| c_122 | Албания | 444.50 |
| c_9 | Египет | 474.50 |
| c_58 | Тунис | 571.00 |
| c_50 | Кипр | 713.50 |

### `GET tour/dates?origin=3164&destination=c_8`
Доступные даты вылета (внимание: здесь параметры в **единственном** числе).

### `GET tour/tours` — главный поиск
Параметры (декодированы из query-builder'а фронтенда):

| Параметр | Формат | Пример |
|---|---|---|
| `origins` | id города вылета | `3164` |
| `destinations` | id стран, **можно списком через запятую** | `c_8,c_4,c_9` |
| `dates` | одна дата или диапазон `от:до` | `2026-07-10:2026-07-25` |
| `stays` | ночей; **надёжно только одно значение**. Список через запятую коварен: 5+ значений → молча пусто; и если хотя бы одно значение без результатов — пуст весь ответ. Причём `tour/stays` может заявлять ночёвку, по которой туров нет | `7` |
| `pax_adl` | взрослых | `2` |
| `pax_chd` | детей (опционально) | `1` |
| `children_ages` | возрасты детей (опционально) | `7` |
| `tour_types` | фильтр-тег (опционально) | `hot_tour`, `super_offer`, `weekly_top`, `t_*` |
| `page` | страница (по 10 туров) | `2` |

Ответ: `tours[]` (сгруппировано по отелям, внутри `offers[]`), `pagination` (`last_page` есть, `total_items` — null), `filters[]` (доступные фильтры: типы туров, питание и т.д.).

**Мульти-направление работает**: все 8 стран одним запросом, выдача сортирована по цене — это ровно наш «поиск без направления». Замечание: `stays` диапазоном `6:8` при мульти-направлении даёт ошибку `stays.0` — слать одно значение (или несколько запросов).

### `GET tour/offers?...&hotel_ids=31891`
Все офферы конкретного отеля (варианты питания/номеров) при тех же параметрах поиска.

### Прочее (не проверено глубоко, есть в коде фронтенда)
`tour/calendar`, `tour/filters`, `tour/regions`, `tour/stays`, `hotel/info`, `hotel/hotels`, `hotel/tripadvisor` (рейтинги TripAdvisor — параметр `ids`, наш вызов вернул 500; разобраться в v3 — возможно, нужен другой id отеля), `booking/check` (там может жить остаток мест).

## Структура оффера (что реально приходит)

```
offer:
  id                 — длинный hex-блоб (опаковый; стабильность не гарантирована —
                       для идентичности использовать связку из спеки, не этот id)
  from               — город вылета (id, name)
  date_start / date_end
  stay               — ночей
  board              — питание: board_type (RO/BB/HB/AI...), name, code
  rooms[]            — code, name, placement ("2AD")
  transports[]       — рейсы туда/обратно: номер, авиакомпания, RIX→AYT,
                       время вылета/прилёта, багаж (ручная/регистрируемая, кг)
  price:
    total_price            — текущая цена (EUR)
    average_history_price  — ⚠️ Join Up сам считает среднюю историческую цену!
                             (в наших пробах null — следить, когда заполняется)
    components, installment_price, ...
  availability        — остаток мест (в пробах null; вероятно, заполняется
                        только при дефиците — проверить на живых горящих)
  stop_sale, is_booking_available
  promotion, badges[] — маркеры («горящий» и т.п.)
  pax_adl, pax_chd, children_ages
hotel:
  id, name, category (звёзды), badges, descriptions, media[] (фото), links
```

Все поля из «правила идентичности оффера» спеки присутствуют: отель + дата вылета + ночи + питание + тип номера + состав + аэропорт вылета.

## Проверенные живые запросы

```bash
# города вылета
curl 'https://joinup.lv/api/main/search/origins?lang=lv&currency=EUR'

# направления из Риги
curl 'https://joinup.lv/api/main/tour/destinations?origins=3164&lang=lv&currency=EUR'

# поиск без направления: все страны, диапазон дат, сортировка по цене
curl 'https://joinup.lv/api/main/tour/tours?origins=3164&destinations=c_8,c_4,c_9,c_3,c_122,c_28,c_58,c_50&pax_adl=2&stays=7&dates=2026-07-10:2026-07-25&lang=lv&currency=EUR'

# только горящие в окне последней минуты
curl '...tour/tours?...&tour_types=hot_tour&dates=2026-07-07:2026-07-20...'

# все офферы одного отеля
curl 'https://joinup.lv/api/main/tour/offers?origins=3164&destinations=c_8&pax_adl=2&stays=7&dates=2026-07-25&hotel_ids=31891&lang=lv&currency=EUR'
```

Живые цифры на 2026-07-06: Рига→Турция, 7 ночей, 2 взрослых, вылет 2026-07-25 — 33 страницы (~330 отелей), от 969 EUR (Ares City 3★, BB, рейс 4M-934 RIX→AYT). Горящие по всем странам на 2 недели вперёд — ~60 офферов, от 833 EUR (Албания).

## Что это значит для роадмапа

- **v0 — лёгкий.** Никакого парсинга HTML: обычные GET-запросы с JSON-ответами.
- **Снимок цены** — поля `total_price`, `availability`, `badges/tour_types` есть в каждом ответе.
- **Бонус:** `average_history_price` — оператор сам отдаёт среднюю цену; можно сверять нашу историю с их цифрой.
- **v3 (отзывы):** у API есть готовый `hotel/tripadvisor` — возможно, отзывы вообще не придётся добывать снаружи.
- **Риск доступа** остаётся: API неофициальный, формат и доступность могут поменяться без предупреждения. Вежливая частота опроса обязательна.
